根因分析

核心问题：Text/TextSpan 不输出颜色 escape

Flitter 的 DensityOrbWidget 使用 StatefulWidget + Text(TextSpan) 模式构建， 每个 TextSpan 都设置了 TextStyle({ foreground: Color.rgb(0,G,B) })， 但 tmux capture 显示没有产生任何 ^[[38;2;R;G;B;m escape 序列。 这说明 渲染管线中 TextStyle.foreground 的 RGB Color 没有被正确转换为 ANSI true-color escape codes。

同样的问题影响了 "Welcome to Amp" (GlowText)、"Ctrl+O" (blue)、"for help" (yellow) 等文本元素 —— 这些在测试中通过了颜色断言（headless rendering 中 Grid 可读取到颜色），但在实际终端中颜色 escape 未输出。

排查方向：flitter-core 的 ScreenBuffer → ANSI output 管线， 检查 Cell.fg (RGB Color) → ^[[38;2;R;G;Bm 的转换是否在真实终端模式下生效。 headless capture (captureToGrid) 可以读取到颜色，但 stdout 输出可能跳过了 true-color escape。

## Issues Found

### Non-blocking

1. **CLAUDE.md stale file path** (line 117): `wB0 | TerminalManager | terminal/terminal-io.ts` should be `terminal/terminal-manager.ts`. The file `terminal-io.ts` does not exist; it was renamed to `terminal-manager.ts` during Phase 23.

2. **ROADMAP.md not updated**: The v1.3 Progress table still shows "0/3 Planned" for all phases. The phase checkboxes in the v1.3 Phases section still show `[ ]` instead of `[x]`. REQUIREMENTS.md traceability still shows "Planned" and "Completed: 0".

3. **Element.markNeedsRebuild uses `require()` instead of static import**: ELEM-01 says "must import properly, not use dynamic require()". The implementation uses `require('./binding')` for circular dependency avoidance. However, there is no try/catch wrapping it (which was the critical anti-pattern), and the call chain is architecturally correct. This is a practical concession to TypeScript/Bun circular import limitations.

4. **InputBridge source file still exists**: `src/input/input-bridge.ts` is still present in the codebase. It is NOT imported by binding.ts or any production code path (only by its own test file). Phase 26 was supposed to "remove dead code: InputBridge standalone usage removed from binding" -- the binding usage IS removed, but the source file and its tests remain. This is non-blocking since it is an unused utility.

5. **MaxListenersExceededWarning**: The test suite emits warnings about "MaxListenersExceededWarning" from `platform.ts:184` (onResize). This is a test isolation issue (process event listeners accumulating across test runs), not a functional bug.

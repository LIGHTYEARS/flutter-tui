# Plan 01: Permission Dialog Widget

<objective>
Create a permission dialog widget that renders when the ACP agent requests permission. The dialog uses flitter-core's SelectionList for option selection and is overlaid on top of the main app content. Selecting an option resolves the pending promise in AppState, unblocking the agent. Pressing Escape cancels.
</objective>

<must_haves>
- Permission dialog appears when agent sends requestPermission
- Dialog shows tool call info (title, kind) and permission options
- SelectionList with j/k/arrows/Enter navigation
- Enter selects option, Escape cancels (resolves null)
- Selection resolves AppState.pendingPermission promise
- Dialog overlays on top of main content (Stack pattern)
</must_haves>

2. In `AppStateWidget.build()`, wrap the existing FocusScope return in a Stack when `appState.hasPendingPermission` is true.

Change the return from the current `return new FocusScope({...})` to:

```typescript
const mainContent = new FocusScope({
  // ... existing code unchanged ...
});

// If permission dialog pending, overlay it
if (appState.hasPendingPermission) {
  const request = appState.permissionRequest!;
  return new Stack({
    children: [
      mainContent,
      new Positioned({
        top: 0, left: 0, right: 0, bottom: 0,
        child: new PermissionDialog({
          request,
          onSelect: (optionId: string) => {
            appState.resolvePermission(optionId);
          },
          onCancel: () => {
            appState.resolvePermission(null);
          },
        }),
      }),
    ],
  });
}

return mainContent;
```

3. In the FocusScope onKey handler, add Escape handling BEFORE the existing shortcuts (it should be the first check):
```typescript
if (event.key === 'Escape') {
  if (appState.hasPendingPermission) {
    appState.resolvePermission(null);
    return 'handled';
  }
  return 'ignored';
}
```

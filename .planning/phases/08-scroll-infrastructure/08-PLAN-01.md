# Phase 8: Scroll Infrastructure — Plan 01

**Wave:** 1 of 1
**Scope:** SCROLL-01 through SCROLL-04

## Tasks

### Task 1: RenderScrollViewport — listen for scroll offset changes (SCROLL-01)

**File:** `packages/flitter-core/src/widgets/scroll-view.ts`

Add `attach()`/`detach()` lifecycle to `RenderScrollViewport`:
- In `attach()`: add a listener on `scrollController` that calls `markNeedsPaint()`
- In `detach()`: remove the listener
- Also listen when `scrollController` setter is called (swap listeners)

This ensures the viewport repaints when the scroll offset changes (keyboard, mouse, or programmatic).

### Task 2: Disable follow mode on upward scroll (SCROLL-02)

**File:** `packages/flitter-core/src/widgets/scroll-view.ts`

In `ScrollableState._handleKey` and `_handleScroll`:
- When scrolling **up** (negative delta, k, ArrowUp, PageUp, Ctrl+u, mouse button 64): call `ctrl.disableFollowMode()` before the scroll action
- When scrolling **down**: do not disable (jumpTo will re-enable at bottom)

### Task 3: Fix stateListener follow mode logic (SCROLL-03)

**File:** `packages/flitter-amp/src/app.ts`

Change the stateListener to only enable follow mode if the scroll was already at bottom:
```typescript
this.stateListener = () => {
  const wasAtBottom = this.scrollController.atBottom;
  this.setState(() => {});
  if (this.widget.appState.isProcessing && wasAtBottom) {
    this.scrollController.enableFollowMode();
  }
};
```

### Task 4: Enable keyboard scroll on chat ScrollView (SCROLL-04)

**File:** `packages/flitter-amp/src/app.ts`

Add `enableKeyboardScroll: true` to the `SingleChildScrollView` constructor.

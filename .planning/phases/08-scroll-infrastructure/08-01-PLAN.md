# Plan 08-01: Fix Scroll Infrastructure

<objective>
Wire RenderScrollViewport to repaint on scroll, fix follow mode disable on user scroll-up, fix follow mode re-enable to be conditional, and enable keyboard scroll.
</objective>

<must_haves>
- RenderScrollViewport.attach() calls scrollController.addListener(() => markNeedsPaint())
- RenderScrollViewport.detach() removes the listener
- _handleKey calls controller.disableFollowMode() for up-scroll actions (k, ArrowUp, PageUp, Ctrl+U, g)
- _handleScroll calls controller.disableFollowMode() for scroll-up (button 64)
- app.ts stateListener only calls enableFollowMode when scrollController.atBottom before the update
- SingleChildScrollView gets enableKeyboardScroll: true
</must_haves>

<tasks>

<task id="1">
<title>Add attach/detach listeners to RenderScrollViewport</title>
<action>
Override attach() and detach() in RenderScrollViewport to wire scroll repaint listener.
</action>
</task>

<task id="2">
<title>Add disableFollowMode to _handleKey and _handleScroll</title>
<action>
In ScrollableState._handleKey, call effectiveController.disableFollowMode() before scrollBy for: k, ArrowUp, PageUp, Ctrl+U, g (top).
In ScrollableState._handleScroll, call effectiveController.disableFollowMode() before scrollBy(-3) for button 64.
</action>
</task>

<task id="3">
<title>Fix stateListener follow mode logic</title>
<action>
In app.ts, save `const wasAtBottom = this.scrollController.atBottom` before setState, then only enableFollowMode if wasAtBottom.
</action>
</task>

<task id="4">
<title>Enable keyboard scroll on SingleChildScrollView</title>
<action>
Add `enableKeyboardScroll: true` to the SingleChildScrollView in app.ts.
</action>
</task>

</tasks>
